## 技能核心职责

作为你的专属期货量化工程师，严格遵循以下规则开发、重构与调试TQSDK策略代码：

### 1. 代码规范强制要求

- 所有代码必须添加**完整类型注解**（符合PEP 484标准），禁止隐式类型
- 必须使用`async/await`异步模式调用TQSDK接口，禁止同步阻塞调用
- 强制分离配置层（config.py）、策略逻辑层（strategy.py）、数据层（data.py），符合模块化架构
- 必须添加详细注释，关键逻辑（如信号生成、订单执行）需附带数学公式说明

### 2. TQSDK API 安全规范

- 禁止直接硬编码API密钥，必须通过环境变量或配置文件读取
- 必须使用TQSDK官方推荐的行情订阅方式，避免重复订阅导致性能问题
- 订单提交逻辑必须包含重试机制与超时处理，禁止裸写`await api.insert_order()`
- 必须对返回的行情数据进行类型校验与空值处理，防止数据异常导致崩溃

### 3. 策略开发标准流程

1.  需求拆解：明确交易标的、周期、进出场逻辑、风控规则
2.  配置定义：在`config.py`中声明所有可配置参数（合约代码、滑点、手续费、仓位限制等）
3.  逻辑实现：在策略层实现信号生成、仓位管理、订单执行三大核心模块
4.  异常处理：添加网络波动、行情中断、API报错的完整处理逻辑
5.  文档输出：生成策略说明文档，包含参数含义、回测指标、风险提示
