# 项目提示词

本项目是日内低频期货交易策略项目，当前核心目标是将天勤 TQSDK 的回测/模拟交易代码逐步转化为可实盘迁移的稳定结构。

## 项目定位

- 默认使用 Python 3.13、Windows 环境、TQSDK。
- 策略类型限定为中国期货市场的日内低频或波段策略。
- 当前优先维护 `config.py` 与 `strategy.py`，除非明确要求，不做大规模目录重构。

## 开发原则

- 所有修改优先采用最小 diff，保留现有结构、命名和配置习惯。
- 不添加未经要求的复杂框架、抽象层或多余功能。
- 修改策略逻辑前先说明假设、风险点和可验证目标。
- 修复 bug 时先定位 root cause，再做局部修改并验证。

## TQSDK 与交易安全

- 禁止硬编码账号、密码、密钥；优先从环境变量或本地私密配置读取。
- 回测、模拟、实盘模式必须有清晰边界，避免误把测试逻辑用于实盘。
- 下单逻辑必须考虑超时、异常、网络中断、行情中断、涨跌停和非交易时段。
- 使用 `TargetPosTask` 或其他下单接口时，必须先经过仓位和风控检查。

## 回测与策略约束

- 严禁未来函数、未来价格引用、跨周期错位和幸存者偏差。
- 回测成交假设必须显式考虑滑点、手续费、保证金和成交延迟。
- 策略指标如 KAMA、VWAP、Hurst、FVG 等，新增或修改时需说明公式和数据窗口。
- 多周期逻辑必须确认时间对齐，不能用尚未完成的高周期 K 线生成当前信号。

## 风控要求

- 必须保留或补充单品种仓位限制、总风险限制、日内亏损熔断、止损止盈保护。
- 不允许满仓、不允许隐式杠杆、不允许无限加仓。
- 极端行情、流动性不足、涨跌停、网络异常时必须优先保护资金安全。

## 验证要求

- 策略修改后优先运行语法检查或可行的本地验证。
- 回测相关修改需检查交易次数、最终权益、PnL 一致性、最大回撤和异常交易点。
- 如果环境缺少 TQSDK 登录、行情权限或依赖，需明确说明无法完整验证的原因。
